1. ГЛАВНАЯ ЦЕЛЬ
================================================================================

Подключить уже подготовленные Workshop-моды Project Zomboid к серверному конфигу.

Нужно:

1. Найти основной конфиг сервера Project Zomboid:
   servertest.ini

2. Сделать резервную копию servertest.ini перед изменением.

3. Взять подготовленные строки WorkshopItems= и Mods= из документа:
   M:\GitHub\ProjectZomboid\ProjectZomboud\Documentation\Список_модов_Project_Zomboid.md

4. Внести эти строки в servertest.ini.

5. Найти и зафиксировать основной файл запуска сервера:
   StartServer64.bat

6. Обновить проектные правила сервера, чтобы было понятно:
   - где основной файл запуска;
   - где основной файл настройки;
   - где лежат моды;
   - какой путь локальный на рабочей машине;
   - какой путь сетевой;
   - какой путь локальный на серверной машине.

7. Сервер в этой задаче не запускать и не останавливать без отдельной явной команды пользователя.

7.2. Найти основной конфиг сервера

Найти файл:

servertest.ini

Обычно он находится в:

C:\Users\<имя_пользователя>\Zomboid\Server\servertest.ini

Если агент работает с рабочей машины и C:\Users серверной машины недоступен напрямую, проверить, есть ли доступный сетевой путь к профилю пользователя сервера. Если доступа нет — остановить правку конфига и указать проблему.

7.3. Сделать резервную копию servertest.ini

Перед изменением обязательно сделать копию рядом с исходным файлом:

servertest.ini.backup_TASK007_YYYY-MM-DD_HH-mm-ss

Если backup создать не удалось — не менять servertest.ini.

7.4. Взять строки WorkshopItems и Mods

Открыть:

M:\GitHub\ProjectZomboid\ProjectZomboud\Documentation\Список_модов_Project_Zomboid.md

Взять оттуда полные строки:

WorkshopItems=...

Mods=...

Не пересобирать вручную, если документ уже содержит актуальные строки.

7.5. Обновить servertest.ini

В найденном servertest.ini найти строки:

WorkshopItems=
Mods=

Если строки уже есть:
- заменить их на подготовленные строки из Список_модов_Project_Zomboid.md.

Если строк нет:
- добавить их в файл аккуратно отдельными строками.

Не менять остальные настройки servertest.ini без необходимости.

Не менять:
- PublicName;
- Password;
- ServerPassword;
- AdminPassword;
- RCONPassword;
- PauseEmpty;
- MaxPlayers;
- Map;
- SpawnRegions;
- SandboxVars;
- настройки мира;
- настройки безопасности;
- настройки игроков.

Если в servertest.ini уже были другие WorkshopItems или Mods, сохранить старые значения в отчёте и в backup.

7.6. Проверить результат

После изменения проверить:

- в servertest.ini есть строка WorkshopItems=;
- в servertest.ini есть строка Mods=;
- строки не обрезаны;
- между ID стоит разделитель ;
- нет лишних кавычек;
- нет пробелов вокруг разделителей, если их не было в исходном стиле;
- файл читается нормально;
- backup создан.
