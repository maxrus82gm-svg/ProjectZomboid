# Правила создания документов

## Назначение

Документ описывает правила создания и обновления Markdown-документов в проекте `Project Zomboid Dedicated Server`.

## Базовые правила

- Документы создаются только по задаче пользователя или активной задаче в `M:\GitHub\ProjectZomboid\ProjectZomboud\Задача_агенту.md`.
- Рабочие документы проекта создаются и меняются только внутри `M:\GitHub\ProjectZomboid\ProjectZomboud`.
- Документы сохраняются в Markdown.
- Кодировка: UTF-8.
- Русский текст должен читаться нормально.
- Использовать настоящие переносы строк.
- Не записывать файл одной строкой.
- Не вставлять текстовую пару backslash+n вместо переносов.
- Не экранировать Windows-пути двойными слэшами.
- Не менять `.obsidian` без разрешения.
- Не менять `.git` без прямой команды пользователя.
- Не менять реальные конфиги агентов без прямой задачи.
- Не менять серверную папку Project Zomboid без отдельной явной задачи пользователя.

## Obsidian links

Если проект использует Obsidian, новые документы связываются через Obsidian links, когда это нужно проекту:

```text
[[Название_документа]]
```

Ссылки должны вести только на рабочие документы внутри `M:\GitHub\ProjectZomboid\ProjectZomboud`.

Запрещены Obsidian-ссылки на источник первичного переноса или на папки вне рабочей базы.

## Источник первичного переноса

Источник первичного переноса:

```text
M:\GitHub\ProjectZomboid\Шаблон_агентской_системы
```

Этот источник используется только как внешний исходный материал для первичного развёртывания или отдельной задачи по обслуживанию шаблона. Он не является рабочей папкой правил проекта.

## Правило единого большого и малого блока

Когда ChatGPT готовит задачу для агента, он обязан выдавать большой блок и малый блок как два отдельных цельных фрагмента.

Большой блок:

- предназначен для вставки в `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА` файла `Задача_агенту.md`;
- должен идти одним непрерывным куском;
- не должен быть разорван пояснениями, комментариями или вторыми кодовыми блоками;
- должен содержать весь scope задачи: цель, контекст, разрешённые файлы, запреты, проверки, формат отчёта и правило очистки `БЛОК 1`.

Малый блок:

- предназначен для отправки агенту в чат;
- должен идти отдельным цельным куском после большого блока;
- не должен смешиваться с большим блоком;
- не должен расширять scope и не должен отменять Git-политику без прямого разрешения пользователя.

Правильный формат ответа ChatGPT при подготовке задачи:

1. Один цельный большой блок для `БЛОК 1`.
2. Один цельный малый блок для чата агенту.

Запрещено выдавать большой блок частями, если пользователь просит готовую задачу для вставки.

## Проверка перед завершением

- Документ находится в разрешённом scope.
- Нет дубля уже существующего документа.
- Markdown читается нормально.
- UTF-8 сохранён.
- Нет текстовой пары backslash+n вместо переносов.
- Нет незаменённых placeholders.
- Нет Obsidian-ссылок на источник первичного переноса.
- Нет случайных путей чужого проекта.
- Windows-пути записаны обычным текстом.
