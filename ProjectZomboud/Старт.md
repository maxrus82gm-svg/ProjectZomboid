## Что читать перед задачей

Перед началом любой задачи агент читает:

```text
M:\GitHub\ProjectZomboid\ProjectZomboud\Старт.md
M:\GitHub\ProjectZomboid\ProjectZomboud\Задача_агенту.md
```

Если задача требует тематических правил, агент читает только нужные документы из:

```text
M:\GitHub\ProjectZomboid\ProjectZomboud\Documentation\Список_правил.md
```

Источник первичного переноса:

```text
M:\GitHub\ProjectZomboid\Шаблон_агентской_системы
```

Этот источник не является рабочей частью Obsidian-графа проекта. В обычных задачах агент не использует документы источника как рабочие правила и не создаёт на него Obsidian-ссылки.

## Правило завершения задачи

Если задача успешно выполнена, агент сначала записывает итоговый отчёт в `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА` файла:

```text
M:\GitHub\ProjectZomboid\ProjectZomboud\Задача_агенту.md
```

Затем агент очищает `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА` до состояния:

```text
Активной задачи нет.
Предыдущая задача выполнена.
Ожидается новая задача от пользователя.
```

Если задача выполнена частично, неуспешно или требует решения пользователя, `БЛОК 1` не очищается, а причина указывается в `PROBLEMS`.

## Правило Git

Git контролирует пользователь.

Политика Git:

```text
Git контролирует пользователь. Агент не делает commit, push, pull, merge, rebase, reset, clean, checkout, switch, создание веток, изменение remote, изменение Git-конфига или переписывание истории без прямой команды пользователя.
```

Агент не делает commit, push, pull, merge, rebase, reset, clean, checkout, switch, создание веток, изменение remote или изменение Git-конфига без прямой команды пользователя.

Read-only Git-команды выполняются только когда текущая задача прямо разрешает такую проверку.

## Правило Obsidian Vault

Настройки `.obsidian` не менять без прямого разрешения пользователя.

## Сервер Project Zomboid

Локальный источник Workshop-модов на рабочей машине:

```text
G:\SteamLibrary\steamapps\workshop\content\108600
```

Эта папка не является сервером. Это источник уже скачанных Workshop-модов для копирования.

Путь для агентки по сети:

```text
\\192.168.0.77\Server\Project Zomboid Dedicated Server
```

Локальный путь этого же сервера на серверной машине:

```text
C:\Server\Project Zomboid Dedicated Server
```

Целевая Workshop-папка сервера по сети:

```text
\\192.168.0.77\Server\Project Zomboid Dedicated Server\steamapps\workshop\content\108600
```

Правило безопасности:

Агент может читать и анализировать сервер Project Zomboid по сетевому пути, но любые изменения внутри серверной папки, запуск, остановка, обновление, удаление файлов, изменение конфигов, модов или сохранений разрешены только по отдельной явной задаче пользователя.

При изменении серверных конфигов сначала читать `[[Documentation/Правила_сервера_Project_Zomboid]]`: для текущего `servertest.ini` с русскими комментариями backup обязателен, а сохранение должно сохранять исходную кодировку Windows-1251, если файл читается в ней корректно.

Workshop-моды копируются только в `steamapps\workshop\content\108600`. Не класть Workshop-моды просто рядом с сервером.
